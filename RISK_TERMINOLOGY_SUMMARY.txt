## 🎯 CHANGE: "Low Risk" → "Normal Risk"

┌──────────────────────────────────────────────────────────────────────────────┐
│ NEW RISK LEVELS                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

🟢 NORMAL RISK (was Low Risk)
   • Health indicators within normal ranges
   • Continue healthy habits
   • Active exercise routine

🟡 MEDIUM RISK
   • Some indicators need attention
   • Follow nutrition plan
   • Moderate exercise

🔴 HIGH RISK
   • Requires immediate attention
   • Consult doctor
   • Gentle exercise only


┌──────────────────────────────────────────────────────────────────────────────┐
│ WHY THE CHANGE?                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

✅ Better Understanding
   "Normal" is clearer than "Low"
   "Low" can be confusing (low = bad?)

✅ Medical Standard
   Aligns with medical terminology
   Professional and clear

✅ Positive Framing
   "Normal" sounds reassuring
   Better user experience

✅ International Standard
   Used by health apps globally
   Easier to translate


┌──────────────────────────────────────────────────────────────────────────────┐
│ WHAT CHANGED                                                                 │
└──────────────────────────────────────────────────────────────────────────────┘

Frontend (dashboard.js):
• Converts "Low" → "Normal" for display
• Shows "NORMAL RISK" badge
• Green color maintained

Backend (diet_engine.py):
• Added "Normal" support
• Backward compatible with "Low"
• No breaking changes

CSS (dashboard.css):
• Added .risk-normal class
• Same green styling
